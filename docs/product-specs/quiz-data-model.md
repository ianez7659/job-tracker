# Daily Interview Quiz — Data Model (Step 1)

## Overview

This document describes the data layer for the Daily Interview Quiz feature.
Step 1 covers: source data location, validation schema, Prisma models, and import/seed procedures.
UI, API routes, and Daily Mission integration are addressed in later steps.

## JSON file structure

```json
{
  "version": 1,
  "category": "web-development",
  "categoryGroup": "Tech",
  "description": "...",
  "questions": [
    {
      "id": "web-dev-html-semantic-001",
      "sourceType": "category_specific",
      "questionType": "role_specific",
      "difficulty": "easy",
      "question": "Why is semantic HTML important?",
      "choices": [
        { "id": "a", "text": "..." },
        { "id": "b", "text": "..." },
        { "id": "c", "text": "..." },
        { "id": "d", "text": "..." }
      ],
      "correctChoiceId": "b",
      "correctExplanation": "...",
      "wrongExplanations": [
        { "choiceId": "a", "explanation": "..." },
        { "choiceId": "c", "explanation": "..." },
        { "choiceId": "d", "explanation": "..." }
      ],
      "tags": ["html", "accessibility"]
    }
  ]
}
```

### correctChoiceId vs correctIndex

Source files use **`correctChoiceId`** (e.g. `"b"`), not a numeric index.
Choices may be **shuffled** when a `DailyQuizSessionItem` is created.
The shuffled index is stored in `correctIndexSnapshot` at session creation time and is stable for the lifetime of that session.

## Category mapping

Quiz JSON files use **hyphen-separated** category strings (e.g. `"web-development"`).
`USER_CATEGORIES` in `src/lib/constants/categories.ts` uses **underscore-separated** values (e.g. `"web_development"`).

**Conversion rule:** replace all `-` with `_` (and vice-versa).

### Unsupported categories

Language Programmes (`esl`, `celpip`) have no quiz content. These are excluded via
`QUIZ_UNSUPPORTED_CATEGORIES` in `src/lib/quiz/categoryMap.ts`.
Quiz sessions are not created for users in these categories.

### Question pool per user category

When a session is created for a user, questions are drawn from three pools in order:

| Pool | Example for `web_development` |
|---|---|
| `common` | All common questions |
| `<group>-common` | `tech-common` questions |
| `<category>` | `web-development` questions |

The helper `getQuizSourceCategories(userCategory)` returns the ordered pool list.

## Prisma models

Three models are added to `prisma/schema.prisma`. No existing models are changed (except adding a `quizSessions` relation to `User`).

### QuizQuestion

Question bank. One row per source question imported from `data/quiz/`.

| Field | Type | Notes |
|---|---|---|
| `id` | `String` (cuid) | Internal DB id |
| `sourceId` | `String` (unique) | Original JSON question id |
| `sourceType` | `String` | `common` / `group_common` / `category_specific` |
| `categoryGroup` | `String` | `Tech` / `Management` / `Marketing` / `Common` |
| `category` | `String` | Hyphen format (e.g. `web-development`) |
| `questionType` | `String` | `behavioral` / `situational` / `role_specific` / `interview_basics` |
| `difficulty` | `String` | `easy` / `medium` / `hard` |
| `question` | `String` | Question text |
| `choices` | `Json` | `[{id, text}, ...]` |
| `correctChoiceId` | `String` | Matches one of the choice ids |
| `correctExplanation` | `String` | |
| `wrongExplanations` | `Json` | `[{choiceId, explanation}, ...]` |
| `tags` | `Json` | `string[]` |
| `isActive` | `Boolean` | Default `true` — inactive questions are excluded from sessions |

### DailyQuizSession

One session per user per day per category.

| Field | Type | Notes |
|---|---|---|
| `id` | `String` (cuid) | |
| `userId` | `String` | References `User.id` |
| `dateKey` | `String` | `YYYY-MM-DD` in user's local timezone |
| `category` | `String` | Underscore format (e.g. `web_development`) |
| `categoryGroup` | `String` | Lowercase parent id (e.g. `tech`) |
| `status` | `String` | `not_started` / `in_progress` / `completed` |
| `totalQuestions` | `Int` | Default 5 |
| `completedQuestions` | `Int` | Incremented as items are answered |
| `completedAt` | `DateTime?` | Set when all questions are answered |

Unique constraint: `(userId, dateKey, category)` — one session per user per day per category.

If a user changes their category later, old sessions retain their original `category` value. No migration of past session data is needed.

### DailyQuizSessionItem

One row per question within a session. All content is **snapshotted at session creation** so the session is stable across refreshes and source data changes.

| Field | Type | Notes |
|---|---|---|
| `id` | `String` (cuid) | |
| `sessionId` | `String` | References `DailyQuizSession.id` |
| `questionId` | `String?` | Nullable — AI-generated items have no `QuizQuestion` row |
| `sourceType` | `String` | |
| `orderIndex` | `Int` | 0-based position in the session |
| `renderedQuestionSnapshot` | `String` | Question text at creation |
| `choicesSnapshot` | `Json` | Shuffled `[{id, text}, ...]` |
| `correctChoiceIdSnapshot` | `String` | Original correct id |
| `correctIndexSnapshot` | `Int` | Index of correct choice in shuffled array |
| `correctExplanationSnapshot` | `String` | |
| `wrongExplanationsSnapshot` | `Json` | |
| `userAnswerChoiceId` | `String?` | Set when user answers |
| `userAnswerIndex` | `Int?` | Index in shuffled array |
| `isCorrect` | `Boolean?` | Computed on answer |
| `answeredAt` | `DateTime?` | |
| `metadata` | `Json?` | Extensible (AI params, etc.) |

## Import / seed

### Validation

Before import, all files are validated by `src/lib/quiz/validation.ts`.
Run validation only (no DB):

```bash
npx tsx src/lib/quiz/validateAll.ts
```

### Dry run

```bash
npx tsx src/lib/quiz/importQuestions.ts
```

Shows the normalized records that would be upserted. No DB changes.

### Actual import (requires user approval)

```bash
npx tsx src/lib/quiz/importQuestions.ts --write
```

Upserts all questions into `QuizQuestion` using `sourceId` as the upsert key.
Safe to re-run — existing rows are updated, new rows are inserted.
